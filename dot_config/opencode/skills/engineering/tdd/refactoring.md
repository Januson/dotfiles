# Refactor Candidates

See Martin Fowler's *Refactoring: Improving the Design of Existing Code* for complete catalog and techniques.

## Rules

- Refactor only when tests are green
- Tests must pass before and after every change
- One refactoring at a time. Run tests between each.
- Test behavior on the public interface. Do not test private code.

## Code Smells

Duplicated code · Long function · Long parameter list · Data clumps · Primitive obsession · Switch statements · Feature envy · Temporary variables · Divergent change · Shotgun surgery · Speculative generality · Comments explaining what · Lazy class · Data classes · Message chains · Middle man · Alternative classes with different interfaces · Incomplete library classes · Refused bequest · Black sheep · Insider trading
